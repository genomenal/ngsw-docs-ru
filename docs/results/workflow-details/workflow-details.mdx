# Детали процесса анализа

Получить подробную информацию о процессе анализа образцов, статусе выполнения стадий анализа, а также
скачать файлы с результатами стадий можно на вкладке "**Детали процесса анализа**" на странице образца:

<p align="center">
<img src={require('/img/rus/18-workflow-details.png').default} width="1000"/>
</p>

Кроме того, вы можете посмотреть детали каждой стадии анализа по отдельности, нажав
на кнопку <img src={require('/img/results/show_details.png').default} width="25"/> в строке этой
стадии в разделе <a href="/results/main/main#workflow">"Процесс анализа"</a> на вкладке "Главное" на
странице образца. Чтобы из деталей определенной стадии перейти к деталям всего процесса анализа образца,
нажмите на <img src={require('/img/rus/show_all_tasks.png').default} width="150"/>.

### Файл(ы) образца {#sample-files}

Вверху вкладки "Детали процесса анализа" можно скачать файл или файлы (для парного секвенирования) образца:
- У образца, загруженного в формате FASTQ или BAM, для скачивания будет доступен файл или файлы (для парного
секвенирования) в формате FASTQ, сжатые в GZIP архив. В случае загрузки образца с чередованием парных прочтений
для скачивания будут доступны первичный и парный файлы после разделения.
- У образца, загруженного в формате VCF или GT, для скачивания будет доступен файл в формате VCF, не сжатый
в архив, с оригинальной версией референсного генома (hg38 или hg19) и оригинальными названиями хромосом.

Файл(ы) опухолевого образца обозначен(ы) <img src={require('/img/rus/tumor_file.png').default} width="20"/>,
а неопухолевого образца - <img src={require('/img/rus/normal_file.png').default} width="20"/>. Справа от
названия файла указан размер файла.

### Сжатие образца {#compress-sample}

В правом верхнем углу вкладки "Детали процесса анализа"
находится кнопка <img src={require('/img/rus/compress_sample.png').default} width="150"/>.
Нажав на эту кнопку, вы увидите форму для сжатия образцов:

<p align="center">
<img src={require('/img/rus/compress_form.png').default} width="400"/>
</p>

- Чтобы удалить файл(ы) образца, оставьте галочку у опции "*Удалить FASTQ файлы*". Опция доступна только
для образцов, загруженных в формате FASTQ или BAM. После удаления файлов невозможны перезапуск всего анализа
и перезапуск начальных стадий анализа "Загрузка, идентификация и проверка", "Проверка качества и очистка" и
"Выравнивание". Кроме того, будет невозможным изменение параметров
разделов ["Контроль качества"](/settings/settings/quality-control-settings)
и ["Выравнивание"](/settings/settings/alignment-settings). Файлы образцов не будут доступны для
скачивания на вкладке "Детали процесса анализа". Если вы хотите восстановить удаленные
FASTQ файлы, обратитесь в поддержку [info@genomenal.com](mailto:info@genomenal.com).
- Чтобы удалить файлы выравнивания образца, оставьте галочку у опции "*Удалить BAM файлы*".
Опция доступна только для образцов, загруженных в формате FASTQ или BAM, для которых успешно
завершилась стадия "Выравнивание". После удаления файлов станет невозможным перезапуск
стадий анализа, использующих файлы выравнивания, такие как "Предварительная обработка для
выявления вариантов", "Вычисление покрытия", "Выявление соматических SNVs/Indels", "Выявление герминальных
SNVs/Indels", "Выявление структурных вариаций", "Выявление вариации числа копий" и "Геномные предсказания".
Также будет невозможным изменение параметров
раздела ["Выявление вариантов"](/settings/settings/variant-discovery-settings) (кроме
настроек [отбрасывания вариантов, не прошедших фильтрацию,](/settings/settings/variant-discovery-settings#allow-filtered-variants)
и [объединения фазовых групп](/settings/settings/variant-discovery-settings#pg-merge-distance)).
Кроме того, на вкладке "Главное" не будут отображаться результаты выравнивания
(секция ["Выравнивания"](/results/main/alignments)), в деталях задач "Выравнивание" и "Рекалибровка качества
нуклеотидов" в разделе "Файлы с результатами" не будут отображаться файлы в форматах BAM и BAI, а также
не будут отображаться треки выравнивания в браузере IGV. Чтобы восстановить файлы выравнивания, перезапустите
стадию "Выравнивание".
- Чтобы удалить проаннотированные варианты из [SNV Viewer](/results/main/snvs-indels/variant-viewer), оставьте галочку у
опции "*Удалить данные SNV Viewer*". Опция доступна только для образцов, для которых успешно
завершилась стадия "Аннотация SNVs/Indels". После удаления данных будет невозможно перезапустить
стадию "Генерация отчётов", а кнопка SNV Viewer не будет отображаться ни
в [секциях с SNVs/Indels](/results/main/snvs-indels/snvs-indels) на вкладке "Главное", ни в строке этого
образца на странице "Образцы", ни для вариантов в отчётах. Чтобы восстановить данные SNV Viewer,
перезапустите стадию "Аннотация SNVs/Indels".
- Чтобы удалить проаннотированные вариации из [CNV Viewer](/results/main/copy-number-variations/cnv-viewer),
оставьте галочку у опции "*Удалить данные CNV Viewer*". Опция доступна только для образцов, для которых успешно
завершилась стадия "Выявление вариации числа копий". После удаления данных кнопка CNV Viewer не будет
отображаться ни в [секциях с вариациями числа копий](/results/main/copy-number-variations/cnvs) на вкладке
"Главное", ни в строке этого образца на странице "Образцы".
Чтобы восстановить данные CNV Viewer, перезапустите стадию "Выявление вариации числа копий".

Оставив галочки у тех опций, которые вы хотите сжать, нажмите
на <img src={require('/img/rus/delete.png').default} width="150"/>.
Сжать можно только те образцы, анализ которых не находится в процессе.

### Детали стадий анализа {#workflow-details}

На вкладке "Детали процесса анализа" находятся стадии, которые включены в анализ образца и выполнение
которых завершилось или находится в процессе:

<p align="center">
<img src={require('/img/rus/workflow_stages.png').default} width="900"/>
</p>

Для каждой стадии справа отображается статус её выполнения:
- Если стадия находится в процессе выполнения, то в её статусе есть
значок <img src={require('/img/results/in_process.png').default} width="22"/> и процентое
значение прогресса её выполнения.
- Если стадия завершена успешно, то она
имеет статус <img src={require('/img/rus/complete.png').default} width="100"/>.
- Если стадия завершена успешно, однако не все метрики удовлетворяют критериям,
то она имеет статус <img src={require('/img/rus/complete_metrics.png').default} width="100"/> с
указанием количества метрик, которые не соответствуют критериям.
- Если стадия завершена с ошибкой, то в её статусе есть
значок <img src={require('/img/results/failed.png').default} width="25"/> и указана конкретная задача, на которой
анализ прекратился.

Любую стадию можно развернуть, чтобы увидеть её детали, нажав на неё:

<p align="center">
<img src={require('/img/rus/stage_details.png').default} width="900"/>
</p>

Внутри каждой стадии находится таблица со следующими колонками:
- «ЗАДАЧА» - название задачи процесса анализа;
- «НАЧАЛО» - время старта выполнения задачи;
- «КОНЕЦ» - время окончания выполнения задачи;
- «СТАТУС»: если задача находится в процессе выполнения, то в её статусе есть
значок <img src={require('/img/results/in_process.png').default} width="22"/> и процентое
значение прогресса её выполнения; если задача завершена успешно, то она
имеет статус <img src={require('/img/results/complete.png').default} width="120"/>;
если задача завершена успешно, однако не все метрики удовлетворяют критериям,
то она имеет статус <img src={require('/img/results/complete_metrics.png').default} width="120"/> с
указанием количества метрик, которые не соответствуют критериям;
если задача завершена с ошибкой, то в её статусе есть
значок <img src={require('/img/results/failed.png').default} width="25"/> и указано название задачи.

Любую задачу можно развернуть, чтобы увидеть её детали, нажав на неё. Для задач модет быть
приведена следующая информация:
- Вывод консоли о процессе выполнения задачи;
- Проблемы верификации (для задачи "Проверка");
- Файлы с результатами выполнения задачи: все файлы доступны для скачивания, а некоторые можно также открыть
в браузере IGV или в таблицах Google;
- Метрики (для задач "Проверка качества" и "Выравнивание");
- Технические детали выполнения задачи (инструмент, параметры);
- Ошибка при обработке, если задача завершилась с ошибкой.
